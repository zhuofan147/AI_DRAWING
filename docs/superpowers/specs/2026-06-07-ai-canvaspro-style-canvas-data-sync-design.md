# AI_DRAWING 画布能力整体设计方案

## 状态

已按用户确认修订：AI_DRAWING 的画布功能整体参考 AI-CanvasPro 的基础画布能力，同时保留 AI_DRAWING 现有页面流程，并打通页面流程与画布流程的数据。

## 设计结论

AI_DRAWING 不是单纯新增一个“项目关系图”，而是要形成：

```text
AI_DRAWING = 短剧页面工作台 + AI-CanvasPro 式基础画布 + 页面/画布双流程数据拉通
```

页面流程和画布流程是同一个短剧项目的两种操作方式：

- 页面流程适合精细编辑：剧本、分集、角色、分镜、预览、设置。
- 画布流程适合空间化创作：自由节点、素材拖拽、任务队列、多视图、导演台、360 全景、3D 机位规划。
- 两种流程共用同一套业务数据和生成 API，不能形成两套项目、两套素材、两套任务。

AI-CanvasPro 是功能形态参考，不作为直接代码基座。AI_DRAWING 只吸收画布交互、节点能力、资源/任务/文件组织方式，并改造成短剧创作专用画布。

## AI-CanvasPro 式基础画布能力

以下能力属于 AI_DRAWING 画布的基础能力，不是高级扩展。

### 1. 常规画布操作

- 自由拖拽节点。
- 画布缩放、平移。
- 空格拖动画布。
- 鼠标滚轮缩放。
- 框选、多选。
- 复制、粘贴、删除。
- 节点对齐、网格显示、网格吸附。
- 小地图。
- 适应画布。
- 右键菜单。
- 连线显示/隐藏。
- 节点折叠/展开。
- 多画布页签，一个项目内可有多个画布页面。

### 2. 剪贴板能力

画布必须支持从系统剪贴板直接复制粘贴内容。

- 粘贴文本：自动创建文本节点。
- 粘贴图片：自动上传/保存图片资源，并创建图像资源节点。
- 粘贴多段文本：按规则创建一个文本节点或多个文本节点。
- 从外部网页、截图工具、图片编辑器复制图片后，可直接粘贴到画布。
- 粘贴出的图片资源进入统一素材体系，页面流程也能使用。
- 粘贴动作记录来源，例如 `clipboard`，便于后续文件管理和资源追踪。

### 3. 节点类型

基础节点应包含：

- 文本节点：剧本片段、提示词、注释、角色设定、场景描述。
- 图像节点：角色图、参考图、首帧、尾帧、场景图、截图。
- 视频节点：镜头视频、参考视频、成片片段。
- 音频节点：旁白、配音、背景音乐、音效。
- 注释节点：Markdown 备注、创作说明、待办。
- 资源节点：上传素材、生成素材、历史素材。
- 分镜脚本节点：镜头列表、分镜表、镜头描述。
- 短剧业务节点：项目、分集、角色、场景、镜头、素材、导出。
- 任务节点：生成任务、失败重试、批量任务。
- 360 全景节点：全景图查看、场景绑定、镜头绑定。
- 3D 导演台节点：场景空间、角色站位、摄像机机位、镜头运动。
- 组节点：把一组节点组织成流程块，支持整组执行。

### 4. 资源与文件能力

画布应具备 AI-CanvasPro 类似的资源管理体验。

- 素材板：集中展示项目内所有文本、图片、视频、音频、全景、生成历史。
- 文件管理：查看上传文件和生成文件。
- 从素材板拖拽资源到画布创建节点。
- 从生成结果拖拽到画布创建新节点。
- 节点右键加入素材库。
- 素材可绑定到项目、分集、角色、场景、镜头。
- 页面流程生成的素材自动进入素材板。
- 画布生成的素材自动回写页面流程。

### 5. 工作流与任务能力

画布应提供 AI-CanvasPro 类似的任务中心和工作流体验，但业务执行仍走 AI_DRAWING 的短剧 API。

- 任务中心：显示待执行、运行中、完成、失败、取消。
- 支持失败重试。
- 支持批量执行选中节点。
- 支持整组执行。
- 支持按分集、镜头、素材类型筛选任务。
- 支持任务完成提示。
- 支持后台任务状态刷新。
- 页面发起的生成任务和画布发起的生成任务进入同一个任务中心。

### 6. 多视图能力

一个项目内至少应支持这些画布视图：

- 流程视图：项目、分集、角色、镜头、素材、导出之间的关系。
- 素材板视图：集中管理所有素材和生成结果。
- 任务中心视图：集中看生成队列和失败重试。
- 导演台视图：按短剧制作方式查看镜头顺序、画面、状态、时长、生成动作。
- 360 全景视图：查看场景全景图，绑定镜头和场景。
- 3D 导演视角：查看场景空间、角色站位、摄像机、镜头运动。
- 镜头机位规划视图：为每个镜头规划焦段、角度、起止机位、运动方式。

这些视图不是不同项目。它们共享同一个项目数据，只保存不同的布局和视图配置。

## 页面流程与画布流程的数据拉通

### 核心原则

业务数据只保留一份。

- 项目数据来自 `projects`。
- 分集数据来自 `episodes`。
- 角色数据来自 `characters`。
- 分镜/镜头数据来自 `shots`。
- 镜头素材来自 `shotAssets`。
- 生成任务进入统一任务系统。
- 画布布局、视图、节点位置、折叠状态、镜头机位配置等保存到画布相关表。

画布不能复制一份业务数据。否则页面和画布会出现状态不一致。

### 页面到画布

- 页面创建项目后，首页项目卡片可直接进入画布。
- 页面创建分集后，画布出现分集节点。
- 页面提取角色后，画布出现角色节点。
- 页面生成镜头后，画布出现镜头节点。
- 页面生成图片、视频、音频后，素材板出现对应资源。
- 页面执行任务后，任务中心同步状态。

### 画布到页面

- 画布创建文本/图片/视频/音频资源后，页面可以在素材选择处使用。
- 画布触发镜头生成后，分镜页能看到生成结果。
- 画布触发视频生成后，预览页能看到视频。
- 画布编辑角色或镜头的业务字段时，必须调用页面同一套 API。
- 画布节点可一键跳转到对应页面。
- 页面可跳转回画布并定位对应节点。

## 数据结构建议

现有 `canvas_layouts` 可继续用于第一版布局保存，但整体能力需要扩展为更完整的画布数据层。

### canvas_views

保存一个项目内的多个画布视图。

```text
canvas_views
- id
- projectId
- name
- type: flow | asset_board | task_center | director | panorama_360 | director_3d | camera_plan
- nodesJson
- edgesJson
- viewportJson
- settingsJson
- createdAt
- updatedAt
```

### canvas_nodes

保存用户手动创建的非业务派生节点，例如文本、注释、剪贴板图片、自由素材节点、组节点。

```text
canvas_nodes
- id
- projectId
- viewId
- type
- title
- contentJson
- linkedEntityType nullable
- linkedEntityId nullable
- positionJson
- sizeJson
- styleJson
- createdAt
- updatedAt
```

业务派生节点不一定要存入 `canvas_nodes`。例如项目、分集、角色、镜头节点可以从业务表派生；只有用户手动创建或覆盖布局的节点需要持久化。

### canvas_assets

统一登记画布上传、剪贴板粘贴、生成历史、外部拖入的素材。

```text
canvas_assets
- id
- projectId
- type: text | image | video | audio | panorama | file
- source: upload | clipboard | generated | imported | page_flow
- fileUrl nullable
- textContent nullable
- metadataJson
- linkedEntityType nullable
- linkedEntityId nullable
- createdAt
- updatedAt
```

如果素材属于镜头结果，仍以 `shotAssets` 为业务事实；`canvas_assets` 负责资源检索、素材板展示和画布引用。

### director_camera_plans

保存导演台和 3D 导演视角中的机位规划。

```text
director_camera_plans
- id
- projectId
- episodeId nullable
- shotId nullable
- sceneId nullable
- cameraName
- focalLength
- angle
- movement
- startTransformJson
- endTransformJson
- duration
- notes
- createdAt
- updatedAt
```

### panorama_scenes

保存 360 全景与场景/镜头绑定。

```text
panorama_scenes
- id
- projectId
- episodeId nullable
- sceneName
- panoramaAssetId
- linkedShotIdsJson
- hotspotJson
- cameraMarksJson
- createdAt
- updatedAt
```

## 画布 UI 布局

AI_DRAWING 画布整体应接近 AI-CanvasPro 的操作结构。

### 顶部

- 项目名称。
- 当前画布视图 Tab。
- 新建视图。
- 保存状态。
- 当前生成队列状态。

### 左侧浮动工具栏

- 添加节点。
- 画布视图。
- 素材板。
- 工作流。
- 文件管理。
- 任务中心。
- 设置。

### 中间画布

- 节点画布。
- 拖拽、缩放、框选、多选、连线。
- 支持粘贴文本/图片。
- 支持拖入文件。

### 右侧属性面板

- 节点详情。
- 业务绑定信息。
- 生成参数。
- 模型选择。
- 操作按钮。
- 机位/全景/导演参数。

### 底部或角落控件

- 小地图。
- 网格开关。
- 适应画布。
- 缩放滑杆。
- 画布比例。

## 3D 导演视角与镜头机位规划

3D 导演视角不是单纯展示 3D 特效，而是短剧镜头规划工具。

必须支持：

- 场景空间基础视图。
- 角色站位。
- 摄像机位置。
- 镜头朝向。
- 焦段/视角。
- 镜头运动：推进、拉远、横移、环绕、跟随、固定。
- 起始机位和结束机位。
- 将机位规划绑定到镜头。
- 从镜头节点打开对应机位规划。
- 从导演台查看整集镜头机位连续性。

第一版可先用简化 3D 场景表达：地面、角色占位、摄像机锥体、路径线。后续再增加真实 3D 场景资产。

## 360 全景节点

360 全景节点属于基础画布节点。

必须支持：

- 粘贴或上传全景图。
- 在节点中查看全景。
- 绑定到场景。
- 绑定到一个或多个镜头。
- 在全景中标记机位点。
- 在全景中标记角色位置或热点。
- 从导演台打开对应全景。
- 未来可接入全景生成模型，但第一版可以先支持全景资源管理和查看。

## 生成能力关系

AI_DRAWING 画布可以拥有 AI-CanvasPro 式节点，但短剧生成动作必须复用 AI_DRAWING 现有生成链路。

例如：

- 文本节点可生成剧本、角色设定、镜头描述。
- 图像节点可生成角色图、场景图、首尾帧、参考图。
- 视频节点可生成镜头视频。
- 音频节点可生成旁白、配音、背景音乐、音效。
- 分镜脚本节点可生成镜头列表。
- 导演台可批量触发镜头补全。
- 任务中心统一追踪这些动作。

所有实际生成仍通过现有 API 或其扩展版本执行，不在画布里重写一套生成逻辑。

## 实施原则

虽然这些能力都属于基础画布能力，但实现必须分批交付，避免一次性改动过大。

建议按以下顺序实现：

1. 首页、页面、画布互跳补齐。
2. 画布基础操作补齐：多视图、节点添加、剪贴板文本/图片粘贴、右键菜单、小地图/网格/缩放统一。
3. 素材板和文件管理。
4. 任务中心和统一任务状态。
5. 导演台视图和镜头时间线。
6. 360 全景节点。
7. 3D 导演视角和镜头机位规划。
8. 组节点、整组执行、工作流复用。

这些是交付顺序，不代表产品重要性分级。

## 验收标准

第一批基础能力完成后，应满足：

- 用户能从首页项目卡片进入画布。
- 用户能从页面进入画布，并从画布回到页面。
- 用户能在画布中添加文本、图片、视频、音频、注释、分镜脚本等节点。
- 用户能从剪贴板粘贴文本和图片到画布。
- 粘贴图片会进入素材板。
- 页面流程生成的素材会出现在素材板。
- 画布流程生成的素材能在页面流程看到。
- 用户能切换流程视图、素材板、任务中心、导演台、360 全景、3D 导演视角。
- 任务中心能看到页面和画布发起的生成任务。
- 导演台能按镜头顺序展示短剧生成状态。
- 360 全景节点能绑定场景和镜头。
- 3D 导演视角能保存镜头机位规划。

## 非目标

- 不直接照搬 AI-CanvasPro 代码。
- 不把 AI_DRAWING 改成完全通用画布工具。
- 不放弃现有页面流程。
- 不做第二套短剧项目数据。
- 不让任意连线直接改变核心业务逻辑，除非后续设计了明确的工作流执行规范。

## 结论

AI_DRAWING 的画布应以 AI-CanvasPro 的基础画布能力为整体参考，但最终形态要服务于短剧创作。页面流程和画布流程必须数据互通：页面是精细编辑入口，画布是空间化创作和任务编排入口，两者共同组成 AI_DRAWING 的完整短剧创作系统。
